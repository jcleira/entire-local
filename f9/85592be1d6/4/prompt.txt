## Root Cause (jsonl.go:113-131)
The first JSONL user entry looks like: `{"message":{"role":"user","content":"the prompt text"}}` — content is a string, not an array. `messageContent.Content` is `[]contentBlock`, so unmarshal fails. The string fallback tries to parse the entire `message` object as a string, which also fails. Result: empty content, entry dropped.
