# Plan: Selected Row — Full-Width Background + Left Border

## Context
After removing the thick left border and adding a background color, two issues remain:
1. The background only covers the text content, not the full row width
2. User wants a left border indicator back, but without causing layout shift when navigating

## Root Cause
Lipgloss `Background()` only colors the actual text. To fill the full row, we need `.Width(contentWidth)` applied at render time (since terminal width isn't known at style-definition time).

## Changes

### 1. `pkg/ui/dashboard/styles.go` (line 88)

**`selectedRowStyle`**: Add back `BorderLeft(true)` + `BorderStyle(ThickBorder())` + `BorderForeground(primary)`. Keep `Background(lipgloss.Color("236"))`. Set `PaddingLeft(1)` (border 2 chars + padding 1 = 3, matching normalRowStyle's PaddingLeft(3)).

```go
selectedRowStyle = lipgloss.NewStyle().
    Background(lipgloss.Color("236")).
    BorderLeft(true).
    BorderStyle(lipgloss.ThickBorder()).
    BorderForeground(primary).
    PaddingLeft(1)
```

No change to `normalRowStyle` (already `PaddingLeft(3)`, so total left space is identical — no layout shift).

### 2. `pkg/ui/dashboard/checkpoint_list.go` (line 54-57)

Apply `.Width(contentWidth)` at render time for both styles so the background fills the full row:

```go
if i == cursor {
    sb.WriteString(selectedRowStyle.Width(contentWidth).Render(block))
} else {
    sb.WriteString(normalRowStyle.Width(contentWidth).Render(block))
}
```

## Verification
- `make build`
- Visual: selected row has full-width dark background + orange left border, no layout shift when navigating
